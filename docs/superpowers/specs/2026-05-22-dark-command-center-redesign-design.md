# 深色冷链指挥舱整体视觉改造设计

## 背景与目标

当前前端是 Vue 2 医疗冷链监控系统，已有完整路由、API、Pinia store、自动刷新和业务页面。此次改造目标是将整体界面升级为现代深色科技风格，同时不改变现有业务逻辑和接口行为。

目标效果：系统看起来像医疗冷链监控指挥舱，适合设备状态、实时数据、阈值、历史趋势和位置监控等场景。视觉需要现代、清晰、有科技感，但不能牺牲后台系统的可读性和操作效率。

## 视觉方向

采用“深色冷链指挥舱”风格：

- 深蓝黑背景，叠加冷色径向光、细网格和低透明度纹理。
- 青蓝作为主操作色，冷绿色表示正常状态，红橙表示告警状态。
- 卡片使用暗色玻璃质感、细边框、轻微外发光和层级阴影。
- 动效保持适度：页面入场、卡片 hover、按钮光晕、告警状态强调。
- 避免过度赛博化，不使用影响表格、表单可读性的强扫描线或高频动画。

## 改造范围

保留现有路由、API、store、校验、定时刷新和页面数据流，主要改造视觉与局部布局。

重点文件：

- `src/styles/theme.css`：重建全局设计变量、公共组件样式、表单/按钮/表格基础风格。
- `src/App.vue`：全局背景、字体、页面基础 reset。
- `src/components/common/AppShell.vue`：顶部框架、导航、用户状态、内容容器。
- 页面组件：登录、设备管理、阈值、实时数据、实时监测、历史数据、实时位置，以及管理员相关页面。
- 公共数据组件：指标卡、趋势图卡、仪表环等，统一暗色面板风格。

## 页面设计

### 登录页

登录页改成“安全接入舱”卡片。保留手机号、验证码、验证码展示、登录按钮和超级管理员提示。视觉上使用深色玻璃卡片、系统说明区、冷色状态装饰和高对比主按钮。

### 设备管理页

顶部统计区改成四个 KPI 卡片，突出可借设备、系统在用、我的设备和告警数量。设备列表改成深色设备面板，包含设备名称、编码、药品、状态徽标、温湿光电指标、路线位置和快捷操作。告警设备使用红橙边缘光和状态色。

### 阈值设置页

表单改成参数控制面板。设备选择、保存按钮保持在工具栏。阈值输入分组排列，输入框聚焦时使用青蓝发光边框。校验逻辑保持不变。

### 实时数据页

保留表格结构，改成暗色数据表。表头高对比，行分隔更细，告警行使用弱红色背景和警示边线。操作按钮保持紧凑，适合后台快速跳转到监测、历史和位置。

### 实时监测页

四个指标卡改成仪表盘式面板。温度、湿度、光照显示更大的数值和阈值范围；位置显示城市和地址。异常指标使用红橙色边框和背景提示。

### 历史数据页

保留已有历史查询、汇总卡、趋势图和异常列表结构。统一暗色容器后，汇总卡作为顶部分析入口，当前快照作为中间状态，趋势图和异常时刻作为主要分析区域。

### 实时位置页

地图功能保持不变。地图外层增加深色边框和冷色光晕，定位信息改成现代信息条，包含城市、地址、经纬度。百度地图自身不强制改样式，只用外部 UI 统一风格。

### 管理员页面

管理员设备借用概览和借用限制页面同步套用深色面板、工具栏、表格、状态徽标和按钮样式，保持和用户侧页面一致。

## 组件与样式策略

- 用 CSS 变量统一颜色、阴影、圆角、边框和渐变。
- 在全局主题中提供通用按钮、输入框、表格、空状态、工具栏和页面面板样式。
- 页面 scoped 样式只处理该页面独有布局，尽量复用全局变量。
- 不引入新 UI 框架，不新增运行时依赖，避免改动构建配置。

## 响应式设计

- 桌面端保持 dashboard 宽屏布局。
- 中等屏幕下 KPI、指标卡、快照卡从四列降为两列。
- 手机端降为单列，工具栏纵向排列，表格保持横向滚动。
- 不隐藏核心操作，优先保持可用性。

## 测试与验收

验收标准：

- `npm run build` 通过。
- 登录、导航、申请/归还设备、保存阈值、实时数据刷新、实时监测、历史查询、位置刷新等现有流程不被破坏。
- 深色主题在所有主要页面一致，无明显浅色残留卡片或不可读文本。
- 告警、正常、禁用、空状态在深色背景下清晰可区分。
- 移动端基础布局可用，表格可横向滚动。
